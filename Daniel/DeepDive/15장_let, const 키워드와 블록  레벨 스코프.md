# let, const 키워드와 블록 레벨 스코프

### var

- var는 중복 선언이 가능
  - -> 먼저 선언한 변수 값이 바뀔 수 있다.
- 오로지 함수의 코드 블록만을 지역 스코프로 인정한다.
  - -> 전역 변수가 중복 선언 될 수 있다.
- 호이스팅 발생
  - -> 할당문 이전에 변수를 참조할 수 있어, undefined 값을 반환한다.

### let

- 같은 스코프 내에서 중복 선언을 허용하지 않는다.
- 블록 레벨 스코프를 따르기 때문에 중복 선언이 될 수 없다.
- 호이스팅이 발생하지 않는 것처럼 동작한다.

### const

- let 키워드와 대부분 동일. (const는 상수)
- 선언과 동시에 초기화 해야한다.
- 호이스팅이 발생하지 않는 것처럼 동작한다.
- 상수이기 때문에 재할당이 금지된다.
  - -> const로 객체를 할당한 경우 객체는 수정이 가능하다.
